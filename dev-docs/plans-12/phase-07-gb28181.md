## 7.2 GB28181 RTP 收发

**接收模式**:
- UDP 被动接收（服务端开端口等待设备推流）
- TCP 被动接收（服务端监听等待设备连接）
- 单端口模式（所有设备共用一个 UDP 端口，通过 SSRC 区分）
- 可配置端口池范围

**发送模式**:
- UDP 主动发送（向平台指定地址推流）
- TCP 主动连接发送

**RTP 封装**:
- PS over RTP（标准 GB28181）
- 支持 4 字节 TCP 头（RTP-over-TCP）和 2 字节头（部分平台变体）

**API**:
```
POST /api/gb28181/rtp/open   — 创建 RTP 接收端口
POST /api/gb28181/rtp/close  — 关闭 RTP 接收端口
POST /api/gb28181/rtp/send/start — 开始向平台推流
POST /api/gb28181/rtp/send/stop  — 停止推流
```

## 7.3 GB28181 SIP 信令对接

**范围**: 最小可用的 SIP UA 实现，支持：
- REGISTER（设备注册）
- INVITE/ACK/BYE（媒体会话建立/释放）
- MESSAGE（目录查询、设备控制）
- SUBSCRIBE/NOTIFY（目录订阅）

**架构**: 独立 crate `cheetah-gb28181-sip`，不依赖协议三段式（SIP 是信令协议，不是媒体协议）。

**优先级**: 最低。建议先实现 7.1 和 7.2（纯媒体层），SIP 信令可通过外部网关对接。
