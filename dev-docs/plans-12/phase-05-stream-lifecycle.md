# Phase 05 — 流生命周期管理

- **状态**: 未开始
- **范围**: 无人观看自动关流、流超时检测与清理
- **完成标准**: 配置超时后无订阅者的流自动释放资源

---

## 5.1 无人观看自动关流

**问题**: 拉流代理或推流源持续占用资源，即使没有任何播放者。

**方案**:
- 引擎层跟踪每个流的订阅者数量
- 当订阅者数降为 0 时启动计时器
- 超时后触发 `on_stream_none_reader` 事件
- 若配置了自动关流，则释放发布者租约并关闭拉流任务

**配置**:
```yaml
global:
  stream_lifecycle:
    no_reader_timeout_ms: 30000   # 无人观看 30 秒后关流，0 = 禁用
    no_reader_action: close       # close | webhook_only | none
```

**实现位置**: `cheetah-engine` 流管理层，新增订阅者计数监控。

---

## 5.2 流超时检测与清理

**问题**: 推流端异常断开但 TCP 连接未正常关闭（如网络中断），流状态残留。

**方案**:
- 每个发布流跟踪最后收到数据的时间
- 超过配置阈值无数据到达时，判定为超时
- 触发清理：释放租约、通知订阅者 EOF、发送 Webhook

**配置**:
```yaml
global:
  stream_lifecycle:
    publish_idle_timeout_ms: 15000  # 推流 15 秒无数据判定超时
```

**实现位置**: `cheetah-engine` 发布者监控，定期检查最后帧时间戳。
